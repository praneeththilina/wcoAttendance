## 2024-05-19 - Centralized Validation Prevents Mass Assignment and Unsafe Error Checking
**Vulnerability:** The client creation and update endpoints (`POST /clients`, `PUT /clients/:id`) lacked centralized validation middleware, relying on manual validation inside the controller. Furthermore, the controller was using unsafe type assertions (`(error as any).code`) in error handling blocks, which could cause runtime crashes if unexpected error types (like `null`) were thrown.
**Learning:** Inconsistent validation placement (some in middleware, some in controllers) increases the risk of mass assignment vulnerabilities if a developer forgets to validate or uses `req.body` directly before validation. Unsafe error typecasting can lead to unhandled exceptions that mask the true error or cause denial of service.
**Prevention:** Always define Zod schemas in a dedicated validator file (`validators/*.validator.ts`) and apply them at the route definition level using the established `validate(schema.shape)` middleware. Use safe runtime checks (`error && typeof error === 'object' && 'code' in error`) instead of type assertions (`as any`) when inspecting caught errors.
