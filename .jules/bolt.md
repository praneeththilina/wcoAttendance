## 2024-10-27 - Parallelize Prisma Seeding Operations
**Learning:** Sequential Prisma operations inside `for...of` loops, such as those often used in database seeders or data migrations (`findFirst` followed by `create`), can introduce significant cumulative latency (N+1 query problem). This is especially true when each query has to await an individual round-trip to the database.
**Action:** When executing multiple independent database queries or writes (like seeding initial data) where the results of one do not depend on the previous, use `Promise.all` mapping over the array instead of a sequential loop. This allows the Prisma engine to execute the operations concurrently, drastically reducing execution time.
